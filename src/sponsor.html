赞助解答

我们经常看到在商业环境中随意部署 Web 服务器后的灾难性结果，只在为时已晚时向社区求助，提供的信息太少，无法解决可能需要很长时间才能解决的问题——一个如果得到适当培训和支持本可以缓解的事件。

我们对赞助的看法是，Caddy 的所有专业和商业用户都预期投资赞助，就像我们期望树枝为其主干和根部提供营养一样。我们的赞助不仅仅是善意的捐赠。它们为公司提供关键价值，同时维持持续维护。

Caddy 的原始作者 Matt Holt 运营 Dyanim, LLC 来维护项目并与全球社区一起指导开发。赞助由 Dyanim, LLC 执行。

这是软件许可证吗？

不。 赞助是独立产品，提供个人、组织和公司的支持、开发和服务培训、营销以及各种项目管理功能。无论赞助状态如何，您对软件的许可证保持不变。

什么是专业服务？

包括任何补充常规项目维护的服务，如上线、培训、演示和自定义开发工作。

我需要上面没有列出的内容。

没问题——联系我们并告诉我们您需要什么！

只有 GitHub 可以支付吗？

不，我们的高级级别可以接受各种支付方式，如发票、ACH、电汇，甚至物理支票如果需要的话。

我们需要批准新供应商吗？

不太可能！我们的赞助订阅由 GitHub 管理，而 GitHub 通常已经是大多数企业的批准供应商。

为什么要赞助？

专业和商业人士有许多开源许可证无法满足的要求，而更广泛社区有许多专有许可证无法满足的要求。例如，公司需要支持和新功能，但没有业务关系就难以提供或开发。

为了弥合这一差距，我们提供赞助，使我们的软件可以保持免费使用，同时为公司提供他们在承诺如此关键的技术选择时需要的保证。

我们还有针对个人的较低级别，他们只想支持项目，但没有公司资金。

我们的软件很有吸引力，因为赞助使其保持完全开放，没有昂贵、付费独占功能来分割和划分社区。相反，每个在个人实验室或学校项目中运行 Caddy 的人都在使用世界上大型企业部署的相同软件版本。这真的很棒，赞助者使其成为可能。

如果我们落入无法预算的类别怎么办？

我们会要求您的产品经理为什么他们在没有支持手段的情况下部署软件。话虽如此，例如一家成熟企业，如果只有 Startup 级别的预算，可以注册 Startup 级别赞助；这总比没有安全网要好。然而，您会有更高的满意度，因为有适当的赞助级别，因为我们可以更好地满足您的需求。

可以一次性赞助吗？

可以！我们称之为捐赠，并怀着感激之情欢迎它们。

赞助更好

仅靠开源许可证，公司在项目规模上都无法持续。必须建立某种业务关系来支持公司的运营以及项目的持续发展。

通常这是通过收费软件或特定功能、购买保险、聘请顾问或指派内部开发人员学习和维护技术来解决的。

赞助提供了一种简单而深刻的解决方案，与替代方案相比有许多优势：

顾问

咨询费很高（$250-500/小时），以抵消该职业的可变性，通常按小时计费，对于试图最小化成本的客户来说可能会造成压力。

顾问的技能水平各不相同，可能是混合技能集，而不是在相关主题方面的专业知识，例如特定 Web 服务器。您也可能需要在开展业务之前批准他们为供应商。

赞助有实惠的固定费率，没有时间硬限制来缓解压力、降低成本。您可以直接访问世界软件专家，并通过已知供应商 GitHub 支付。

保险

赞助不是保险——它们不是事故发生时退款，也不能合法归类为任何类型的承保。相反，它们通过综合集成策略帮助减少事件的可能性和规模。我们会了解您的设置，帮助您在最佳方式下部署。

与保险合作很繁琐且昂贵，但赞助要容易得多，文件和法律 complication 更少。

内部招聘

数学很简单：熟练的内部开发人员薪水可能为每年 180,000 美元。赞助低一或两个数量级，通常每年 3,000-12,000 美元，并让您访问世界专家。

赞助绕过人力资源和招聘流程，并且由于能够培训现有员工，您仍然可以将这些知识保留在内部。

如果招聘人员离开您的公司，知识往往也会消失。赞助确保技能可以通过人员流动继续存在于您的组织中。

什么都不做

不言而喻，直接访问您依赖的软件维护者 比没有支持、没有安全网、对项目优先级没有影响力要好。

使用开源软件，您只能靠自己。您可以向社区求助，但您不会拥有可以调整维护者优先级的高关注关系。

具体业务成果

赞助帮助您保持竞争优势、降低成本，甚至可能提高客户忠诚度。您将能够满足纯开源下您从未想过可能的要求。
